# Materiabill Roadmap — Tasks (Web Phase)

One row per buildable task. The orchestrator picks **any unblocked task** (all
`Depends On` complete) up to `maxConcurrency` — there are no waves or fixed merge
order. The dependency DAG is in `dependencies.mmd`.

**First task:** `T000` is essential and must complete before product work starts.
It uses `bootstrap.md` as its context and creates scaffolding only — **all
business behavior starts after bootstrap.**

## Conventions

- **Done:** `[ ]` open · `[~]` in progress · `[x]` merged.
- **Size:** `S` (a few files), `M` (a module slice / backend or admin side of a
  domain), `L` (a whole module or cross-cutting concern).
- **Branch:** `feat/tNNN-<slug>` for feature work; bootstrap may use
  `chore/t000-project-bootstrap`; one PR per task.
- **Context:** planning docs an agent should read before starting. Paths are
  relative to `Planning/`. The integration contract
  (`context/technical/inframodern-integration.md`) and architecture/tech-stack
  docs are assumed background for **every** backend task.
- Backend tasks: TDD (tests first, confirm with user). Synced Inframodern data is
  read-only `*_refs`; never mint local IDs for it. Money = integer minor units +
  explicit ISO currency. Every operational table carries `workspace_id`.

## Open Product Decisions (block the tasks they touch — confirm before build)

These are flagged in `context/technical/inframodern-integration.md` §7 and must be
resolved with product before the dependent task starts:

1. **Client identity source (T016).** **Materiabill-native ClientIdentity**
   (verified email/phone — recommended for v1) vs **Inframodern customer
   projection** (consume `inframodern.customers` into `customer_refs`). Both
   business docs agree on native for v1; the legacy "A/B" letters were swapped
   between them, so options are named here, not lettered.
2. **Measurement-unit source (T032).** Consume `inframodern.measurement-units`
   (recommended) vs a local enum.
3. **Tax / VAT / ZATCA in v1 (T018/T023/T039).** If v1 issues VAT invoices, wire
   `tax_refs`; otherwise defer the consumer.
